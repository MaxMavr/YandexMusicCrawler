## Запуск у себя

Проект состоит из краулера, который собирает артистов Яндекс Музыки в PostgreSQL, и
веб-интерфейса на Flask. Запускаются вместе из [`main.py`](main.py).

### 1. Клонирование репозитория
Клонируйте репозиторий с **GitHub**:
```bash
git clone https://github.com/MaxMavr/YandexMusicCrawler.git
cd YandexMusicCrawler
```

### 2. Установка зависимостей

```bash
python -m venv venv && source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Создание таблицы

```bash
createdb music                                   # базу нужно создать один раз
```

### 4. Настройка файла окружения

В репозитории присутствует файл [`env`](env) — это шаблон для создания файла `.env`.
Создайте файл `.env` в корневой директории проекта, далее необходимо заполнить две обязательные переменные:

- Ваш OAuth-токен от **Яндекс Музыки**:
```env
YANDEX_TOKEN=ВАШ ТОКЕН ЯНДЕКС
```
Его можно получить [по этой инструкции](https://yandex-music.readthedocs.io/en/main/token.html).

- Ваш пароль пользователя postgres, который был задан при установке PostgreSQL:
```env
POSTGRESQL_PASSWORD=ПАРОЛЬ ОТ POSTGRESQL
```

### 5. Запуск

```bash
python main.py
```

Краулер стартует в фоне, веб-интерфейс — на *127.0.0.1:5000*. Таблицы создаются
автоматически. Стратегию обхода и параметры можно менять в [`config.py`](config.py).

## Послесловие
- Убедитесь, что у вас есть активная подписка _Яндекс Плюс_ для доступа к артистам
- Используйте программу только в личных целях в соответствии с условиями использования **Яндекс Музыки**

При возникновении проблем обращайтесь, всегда рад поправить свой кодикс <._.>
